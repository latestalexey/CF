ОБЩЕЕ ОПИСАНИЕ.

Конфигурация позволяет:

1. вести учет поступлений и расходов денежных средств

2. долгов полученных и выданных

3. кредитов полученных

4. формировать бюджеты движений денежных средств

Начнем с конца.

Бюджет движения денежных средств

Бюджет ДДС формируется документом "Бюджет", расположен в разделе "Планирование".

При открытии документа заполняется дерево статей ДДС на его форме из справочника "Статьи доходов и расходов".

Для корректной работы документа "Бюджет" необходимо правильно заполнить справочник "Статьи доходов и расходов".

В корне справочника надо создать как минимум 2 группы. Это "Доходы" и "Расходы". Обязательно надо указать реквизит "Знак движения", для доходов - "1", для расходов - "-1". Реквизит "Порядок" нужен для сортировки групп статей в порядке, отличном от сортировки по наименованию.

Реквизит "Знак движения" необходимо указывать только для групп самого верхнего уровня, для более детальных групп и элементов он не имеет значения.

Иерархия справочника не ограничена, т.е. можно создавать группы любого уровня вложенности.

После того, как минимально необходимое количество групп создано, внутри этих групп необходимо создать элементы справочника, по видам движений денежных средств.

Заполнение реквизитов элемента справочника:

1. Порядок - не имеет значения

2. Срок накопления - для информации о сбережении

3. Сумма накопления - для информации о сбережении

4. Статья возврата сбережения - необходим, если этот элемент характеризует статью сбережения, требуется для корректной работы документа "Перемещение средств" со включенным флагом "Отразить в сбережениях"

5. Статья сбережения - необходим, если этот элемент характеризует статью возврата сбережения, т.е. по которой будет отражено поступление сбережения из кошелька сбережения в кошелек доступных средств, требуется для корректной работы документа "Перемещение средств" со включенным флагом "Отразить в сбережениях"

6. Статья родитель - ссылка на элемент этого же справочника. используется в том случае, если ведется раздельный учет расходов: Расходы и Расходы за счет сбережений. в реквизит помещается ссылка на оригинальную статью расхода при создании статьи расхода за счет сбережений. в отчете Бюджет ДДС можно управлять флагом отображения Расходов за счет сбережений - показывать их отдельно или в общей сумме расходов.

Справочник Статьи доходов и расходов.

У групп справочника есть реквизит "Вид дохода расхода", он может быть заполнен только у групп верхнего уровня, на более глубоких уровнях его заполнять не обязательно - он там не учитывается при расчете остатка средств.

Вне групп верхнего уровня элементы создавать нельзя, остатки будут считаться неправильно, потому что только у групп самого верхнего уровня проверяется реквизит "Вид дохода расхода" и на основании значения которого выполняется сложение или вычитание суммы по строке из остатка.

Если это накопление, то можно указать итоговую сумму и срок - они используются в отчете Накопления для более детального отражения состояния накоплений.

 

Основной документ - "Планирование (год)".

На закладке "План" отображается дерево значений, идентичное структуре справочника "Статьи доходов и расходов". Глубина иерархии не ограничена.
В это дерево необходимо внести плановые данные о доходах и расходах.
Ввод данных разрешен только в строки, где поле "Статья ДР" является элементом.
При вводе данных пересчитываются итоги по группам, итоги по строке для колонки Итого и остатки средств по месяцам.
Если включен флаг "Показывать факт", то фактические данные буду отображены в колонках "Факт".
Эти данные берутся из регистра накопления БДР за период от начала года до конца года, год определяется по дате документа. Редактирование колонок "Факт" разрешено, но данные не сохраняются при записи документа. Может быть удобно для просмотра "что будет, если факт...." в форме документа.
Также рассчитывается выполнение плана, в процентах, и отображается в колонках "%".

Начальный остаток средств для плана и факта один и тот же.
Он берется из регистра "Денежные средства" на начало года по дате документа по всем дебетовым кошелькам-виды кошельков "Наличные", "Дебетовая карта" и "Электронные деньги". Ввести начальный остаток можно документом "Ввод остатков средств".
Также начальный остаток можно ввести непосредственно в документ. Если начальный остаток сохранен в документе, он будет использоваться при открытии вместо вычисления остатка по регистру "Денежные средства".

Документ можно распечатать.

В документе можно обновить факт, если были введены какие-то документы, изменившие бюджет, в то время как документ планирования был открыт. Для этого предназначена кнопка "Обновить факт". После обновления все группы будут свернуты. Это происходит из-за передачи дерева значений на сервер и обратно.

На закладке "Сервис" находятся кнопки для заполнения плановых данных в документе по фактическим.

При проведении документа в регистр БДР записываются плановые данные по выбранному варианту. Вариант можно не указывать, но в таком случает нельзя будет сравнить бюджеты обработкой "Сравнение бюджетов".

Для плана можно ввести детализацию - кнопка "Детализация". Это может быть пояснение к сумме, или расшифровка суммы по более детальной аналитике.

Для любой статьи можно посмотреть расшифровку движений по документам - кнопка "Расшифровка".

Можно включать/выключать видимость колонок с данными за предыдущими месяцами относительно текущего (с начала года) - флажок "Скрывать прошлые месяцы".

Можно добавить еще один уровень в дерево - документы движения. Для этого предназначена кнопка "Показать документы".

Кнопка "Врем. итог" показывает сумму плана или факта по статье с начала года то того месяца, который выбран в дереве значений (текущая колонка).

Кнопка "Перенести" выполняет перенос плана на другой месяц, с детализацией (если введена).

Обработка "Сравнение бюджетов". (подсистема Планирование)

Данная обработка предназначения для сравнения двух вариантов бюджетов. Позволяет сравнить в двух вариантах - вертикальном и горизонтальном.

В вертикальном варианте в дереве отображаются по 2 колонки для каждого месяца плана

В горизонтальном варианта - по 2 строки с одним элементом справочника, но с разными вариантами (только для строк с элементами, группы не дублируются)

 

КРЕДИТЫ.

Взятие кредита оформляется документом "Поступление средств".

кошелек - в который помещается сумма кредита.

Статью ДР можно не указывать - не пройдет по бюджету.

Если кошелек - это кредитная карта (овердрафт), то надо заполнить в кошельке сумму лимита, если документом вводится сумма меньшая, чем лимит овердрафта, то это значит, что на момент введения остатка по карте уже был расход (взятие кредита) и сформируется движение приход по регистру Кредиты на сумму разницы между лимитом и суммой в документе - оно будет означать , что у нас есть задолженность по кредиту.

Возврат кредита, в т.ч. и овердрафта, осуществляется документом "Перемещение средств". Если кошелек получатель - кредитная карта, то будет сформировано движение расход по регистру Кредиты на сумму в строке документа. Если статья не выбрана - не отразится в бюджете.

Валюта.

Валюта учитывается по курсу приобретения, если это было приобретение.

Начисленные проценты на депозиты в валюте учитываются:

а)на день получения депозита по курсу банка, если это был простой процент, и если проценты или их часть выдаются в рублях

б)по курсу ЦБ, если проценты

б)на день начисления

или купоны на еврооблигации
